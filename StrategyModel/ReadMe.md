用法：针对一组算法，将每一个算法封装到具有共同接口的独立的类中，从而使他们可以互相替换
理解：策略模式是对算法的包装，把使用算法的责任和算法本省分隔开，委派给不同的对象管理；策略模式通常把一个系列的算法包装到一系列的策略类里面，作为一个抽象策略类的子类，简述就是=>准备一组算法，并将每一个算法封装起	来，使得他们可以互换
算法包含角色：
	*环境角色（Context）：持有一个Strategy类
	*抽象策略角色（Strategy）：这是一个抽象角色，通常有一个接口或抽象类实现，侧角色给出所有的具体策略类所需的接口
	*具体策略角色（ConcreteStrategy）：包装了相关下的算法或行为
